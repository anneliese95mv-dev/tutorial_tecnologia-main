1. Justificación del framework

¿Qué es React?

React es una biblioteca de JavaScript desarrollada por Meta para crear interfaces de usuario interactivas-páginas donde la pantalla cambia sin recargar .

  • Permite desarrollar aplicaciones web modernas.
  • Se utiliza en tiendas en línea, redes sociales y paneles administrativos.
  • Trabaja junto con HTML (JSX), CSS y JavaScript.
  • Se recomienda tener conocimientos básicos de HTML, CSS y JavaScript.

2. Instalación

¿Se precisa instalación? Sí, necesitas Node.js instalado en tu máquina (React corre sobre su motor para el entorno de desarrollo).


            # 1. Instalar Node.js (incluye npm) desde nodejs.org
            node -v   # verificar instalación

            # 2. Crear proyecto con Vite (más rápido que Create React App, que ya está obsoleto)
            npm create vite@latest mi-proyecto -- --template react

            # 3. Entrar e instalar dependencias
            cd mi-proyecto
            npm install

            # 4. Correr en modo desarrollo
            npm run dev

Versiones: Node.js 18+ recomendado. React 18 o 19 (la actual).

Rutas a considerar:

  • src/ → donde vive tu código
  • src/main.jsx → punto de entrada
  • src/App.jsx → componente raíz
  • public/ → archivos estáticos (imágenes, favicon)

    4. Utilización del framework

    Crear una aplicación base

    En React, una aplicación se construye mediante componentes. Un componente es una función de JavaScript que devuelve JSX, una sintaxis similar a HTML que React utiliza para describir la interfaz de usuario. 

    function Saludo({ nombre }) {
        return <p>Hola, {nombre}</p>;
        }

        function App() {
        return (
            <div>
            <Saludo nombre="Sara" />
            <Saludo nombre="Cliente" />
            </div>
        );
        }

    Los componentes pueden reutilizarse y combinarse entre sí como si fueran bloques de construcción. En este ejemplo, el componente App utiliza dos veces el componente Saludo, enviándole diferentes valores mediante propiedades (props).

    Visualizar los resultados

    Para ejecutar la aplicación se utiliza el siguiente comando: 

    npm run dev

    Este comando inicia un servidor de desarrollo local con Hot Reload. Cada vez que se guarda un cambio en el código, el navegador actualiza automáticamente la aplicación sin necesidad de recargar la página manualmente, lo que facilita y agiliza el proceso de desarrollo.

    5. Funcionamiento

    ¿Cómo funciona React?

    React trabaja mediante componentes, que son piezas reutilizables de la interfaz de usuario. Cada componente normalmente se desarrolla en un archivo con extensión .jsx y puede utilizarse en diferentes partes de la aplicación. A diferencia de un sitio web tradicional en HTML, donde normalmente cada página corresponde a un archivo, en React cada archivo representa una parte específica de la interfaz.

    ¿Cómo se editan los archivos?

    Los componentes se crean y modifican directamente desde un editor de código como Visual Studio Code. Posteriormente, se exportan e importan para ser utilizados en otros componentes de la aplicación. 

    // Boton.jsx
        export function Boton() {
        return <button>Click</button>;
        }

        // App.jsx
        import { Boton } from './Boton';

        function App() {
        return (
            <div>
            <Boton />
            </div>
        );
        }

    ¿Cómo se conecta con otras aplicaciones?

    En una aplicación React, los componentes se comunican entre sí mediante import. Cuando es necesario obtener información de otros sistemas, como una base de datos, un servidor o una API externa, React utiliza solicitudes HTTP mediante funciones como fetch() o bibliotecas como Axios. De esta manera, React consume los datos proporcionados por otros servicios sin importar directamente el código de esas aplicaciones.

    7. Ejercicios prácticos

    Ejercicio 1: Crear un componente de bienvenida

    Crea un componente llamado Bienvenida.jsx que reciba el nombre de una persona mediante una propiedad (prop) y muestre un mensaje. 

    function Bienvenida({ nombre }) {
        return <h2>¡Bienvenido, {nombre}!</h2>;
        }

        function App() {
        return <Bienvenida nombre="Sara" />;
        }

    Resultado esperado:

    En el navegador deberá aparecer el siguiente mensaje:

    ¡Bienvenido, Sara!

    Ejercicio 2: Crear un contador

    Utiliza el Hook useState para crear un botón que aumente un contador cada vez que el usuario haga clic. 

    import { useState } from "react";

        function Contador() {

        const [contador, setContador] = useState(0);

        return (
            <div>
            <p>Contador: {contador}</p>

            <button onClick={() => setContador(contador + 1)}>
                Incrementar
            </button>
            </div>
        );

        }

        export default Contador;

    Resultado esperado:

    Al hacer clic en el botón, el número del contador aumentará en uno.

    Ejercicio 3: Mostrar una lista

    Utiliza el método map() para recorrer un arreglo y mostrar cada elemento en pantalla. 

    function ListaFrutas() {

        const frutas = ["Manzana", "Pera", "Naranja"];

        return (
            <ul>
            {frutas.map((fruta, index) => (
                <li key={index}>{fruta}</li>
            ))}
            </ul>
        );

        }

        export default ListaFrutas;

    Resultado esperado:

    La aplicación mostrará la siguiente lista:

  • Manzana
  • Pera
  • Naranja

    8. Conclusiones

    React es una herramienta muy útil para crear páginas y aplicaciones web que necesitan actualizarse constantemente sin recargar toda la pantalla, como ocurre en sistemas de recomendación o formularios interactivos. Su principal ventaja es que organiza el código en partes reutilizables llamadas componentes, lo que facilita su mantenimiento y escalabilidad conforme el proyecto crece. Aunque requiere cierto tiempo de aprendizaje para quienes recién empiezan, una vez comprendido su funcionamiento permite desarrollar interfaces más rápidas, ordenadas y fáciles de actualizar, por lo que se considera una opción adecuada para proyectos donde la interacción con el usuario y la actualización de datos en tiempo real son fundamentales.
